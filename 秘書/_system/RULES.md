# 秘書プロジェクト 運用ルール

## タスク管理ルール

### 1. Notion DB連携
- タスクが発生した場合、Notion MCP（`mcp__8dc76dbb`系）を使ってAkiraのタスクDBに書き出す
- DB名: **諸岡タスクDB**
- データソース: `collection://1c7af136-accb-8091-94e7-000b2f657826`
- 主要プロパティ:
  - `Task name` (title) — タスク名
  - `Status` — Not started / In progress / Done
  - `Priority` — High / Medium / Low
  - `優先度` — 最高🔥🔥 / 高🔥 / 中 / 低
  - `Due date` — 期限
  - `Description` — 説明
  - `チケット種別` — ユーザーストーリー🙍 / タスク🔨 / バグ🐞 / エピック
  - `Sprint` — Sprint 1〜52

### 2. ローカルタスク記録
- 内部的に発生したタスク（派生タスク、気づき等）は `_tasks/ACTIVE.md` に記録する
- 完了したタスクは `_tasks/ARCHIVE.md` に移動する
- Notion DBへの書き出しが完了したらローカル側にもNotionリンクを記載する

### 3. 作業ログ・引き継ぎノート（_logs/）
- タスクを1つ終えるごとに、`_logs/` にログを残す
- ファイル名は `YYYY-MM-DD_トピック名.md` 形式
- 記録すべき内容：
  - **やったこと** — 作業内容の要約
  - **判断・思考メモ** — なぜその方法を選んだか、迷ったポイント、気づき
  - **引き継ぎ事項** — 次のセッションや次のタスクで知っておくべきこと
  - **未解決・保留** — 残っている課題、確認待ちの事項
- このログは次回セッションの秘書が「前任の引き継ぎノート」として読む前提で書く

## ファイル管理ルール

### ディレクトリ構成
```
秘書/
├── _system/       # システム設定（このファイル等）
├── _tasks/        # タスクのローカル記録
├── _logs/         # 作業ログ・引き継ぎノート
├── outputs/       # 成果物の保存先
└── （ルート）      # 既存ファイル・一時ファイル
```

### 成果物
- 作成した文書・資料は `outputs/` に保存する
- ファイル名は内容がわかる日本語名を推奨
