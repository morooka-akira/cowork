# AI論文 深掘り対談ポッドキャスト

> 仮称: 未定（要ネーミング）
> ステータス: アイデア段階 → プロトタイプ制作へ
> 起案日: 2026-04-11

---

## コンセプト

AI・技術系の論文を、かわいい女の子2人の対談形式で深掘り解説するポッドキャスト。
NotebookLMのような「表面的な自動要約」ではなく、**構成設計されたプロンプト＋キュレーション**で「聞いてて面白い」クオリティを実現する。

## なぜやるか

- Akira自身がユーザーになれる（自分が聞きたいものを作る）
- NotebookLMは既定フォーマットで深掘りが浅い。その上を狙える
- 最終的にパイプラインを横展開し、スケーラブルなビジネスに育てる

## フォーマット

| 要素 | 内容 |
|------|------|
| 形式 | 対談ポッドキャスト（音声のみ） |
| キャラ構成 | 質問役（引き立て）＋ 専門家役の2名。両方女性キャラ |
| 音声 | AIっぽくない、自然でかわいい女性ボイス |
| 1話の長さ | 未定（10〜20分を想定） |
| 配信先 | Spotify等のポッドキャストプラットフォーム |

## 対談の構造

```
質問役: 「今日はどんな論文？」
専門家役: 論文の概要を紹介
質問役: 「それって何がすごいの？」（素朴な疑問で深掘り）
専門家役: 本質的な新しさを解説
質問役: 「実際のビジネスだとどう使えるの？」
専門家役: 実務への示唆・応用可能性
質問役: 「じゃあ限界とか課題は？」
専門家役: 論文の限界・今後の展望
```

## ターゲット

- ビジネスパーソン（AI活用に関心があるが論文は読まない層）
- エンジニア（論文を追いたいが時間がない層）
- 共通ニーズ: 通勤中・移動中に「今週押さえておくべきAIの動き」をキャッチアップしたい

## 競合・先行事例

### 直接競合（AI論文 × キャラ音声）
- **AIcia Solid Project** — 登録者5.8万人。データサイエンスVTuber。論文・数式解説。最も近い存在
- **ずんだもん + arXiv解説** — 個人開発者レベルで散発的に存在。専門チャンネルとしては未確立

### 間接競合
- **Google NotebookLM** — 論文PDF→AI対談を自動生成。無料。ただし深掘りが浅い
- **Two Minute Papers** — 英語圏最大（177万人）。人間ナレーション。かわいい声ではない
- **Best AI Papers Explained** — 英語ポッドキャスト。プロフェッショナル路線

### 差別化ポイント
- NotebookLM: 自動生成で浅い → こちらは**構成設計された深掘り**
- AIcia Solid: VTuber形式で動画 → こちらは**ポッドキャスト（ながら聴き最適化）**
- ずんだもん系: VOICEVOX音質 → こちらは**より自然でかわいい音声**を追求

## ビジネスモデル（段階的）

### Phase 0: プロトタイプ（現在地）
- Akiraが手動で数本制作
- 「聞いてて面白い」のクオリティ基準を確立
- 技術スタック（音声合成・台本生成）の検証

### Phase 1: ポッドキャスト運営
- Spotify等で定期配信
- リスナー獲得・フィードバック収集
- 台本生成パイプラインの半自動化

### Phase 2: パイプラインの成熟
- 「論文→構成設計→台本→音声→配信」の自動化率を上げる
- Akiraの関与を「選定＋クオリティチェック」に絞る
- プロンプト設計・構成アルゴリズムがコア資産になる

### Phase 3: 横展開（本命）
- 磨いたパイプラインを他ジャンルに展開
  - 医療論文、法律、ビジネスニュース、業界レポート等
- B2B提供（企業の社内教育、ナレッジ共有）
- SaaS化 or ライセンス販売
- **ここがスケーラビリティの本体**

## リスク・未解決の問い

1. **「論文から構成を考えるアルゴリズム」の難易度** — LLMプロンプトでどこまで深掘りの質を担保できるか？ 手動との差はどれくらい出るか？
2. **音声合成の選定** — VOICEVOX（無料・キャラ音声）vs ElevenLabs（高品質・有料）vs Style-Bert-VITS2（カスタム可能）。コストと品質のバランス
3. **ポッドキャスト単体のマネタイズ天井** — Phase 3に到達するまでの間、収益ゼロでも続けられるか？
4. **横展開時の参入障壁** — パイプラインを作っても、他社がLLM＋音声合成で同じことをやるのは容易。Moatは何か？
5. **著作権・論文利用の法的整理** — 論文の内容を要約・解説する際の権利関係
