## Playwright MCP ツールリファレンス

このスキルで主に使用するツール：

| ツール | 用途 |
|---|---|
| `browser_navigate` | URLへの遷移 |
| `browser_snapshot` | アクセシビリティツリー取得（要素の確認・参照取得） |
| `browser_screenshot` | 画面キャプチャ（目視確認が必要な場合） |
| `browser_click` | 要素のクリック（ref または テキスト指定） |
| `browser_type` | テキスト入力フィールドへの入力 |
| `browser_evaluate` | JavaScript実行（DOM操作・選択肢クリック） |
| `browser_tab_list` | 開いているタブ一覧の取得 |
| `browser_tab_select` | タブの切り替え |
| `browser_wait_for` | セレクタ/ネットワーク待機 |
| `browser_select_option` | ドロップダウン選択 |

**基本方針**: まず `browser_snapshot` でアクセシビリティツリーを取得し、要素の `ref` を取得してから `browser_click` でクリックする。DOMが複雑で `ref` が使いにくい場合は `browser_evaluate` でJSクリックを使う。

## Step 2: アンケートを開く

### 「回答する」ボタンのクリックと新タブ対応

マクロミルの「回答する」ボタンはクリックすると**新タブが開く**。Playwright MCP での対応手順：

1. **クリック前にタブ一覧を取得**:
   ```
   browser_tab_list
   ```
   → 現在のタブ数とIDを記録する

2. **「回答する」ボタンをクリック**:
   - まず `browser_snapshot` で「回答する」ボタンの `ref` を取得
   - `browser_click` でクリック（`ref` が機能しない場合は `browser_evaluate` でJSクリック）
   
   ```javascript
   // JSクリックのフォールバック
   Array.from(document.querySelectorAll('a, button, input')).find(el =>
     el.textContent?.includes('回答する') || el.value?.includes('回答する')
   )?.click()
   ```

3. **新タブへの切り替え**:
   ```
   browser_tab_list
   ```
   → クリック前と比べて増えた新しいタブのIDを特定
   ```
   browser_tab_select: {tabId: 新しいタブのID}
   ```

### 同意画面の操作

アンケートを開くと同意確認画面が表示される。

1. `browser_snapshot` で「同意して回答する」ボタンの `ref` を取得
2. `browser_click` でクリック（`ref` で動作しない場合は JS）:
   ```javascript
   Array.from(document.querySelectorAll('input')).find(el => el.value === '同意して回答する')?.click()
   // または
   Array.from(document.querySelectorAll('button')).find(el => el.textContent?.includes('同意して回答する'))?.click()
   ```
3. `browser_wait_for` または 2〜3秒後に `browser_snapshot` で設問画面に遷移したことを確認

## Step 3: 設問への回答方針

### モデル切り替えルール（重要）

このスキルは **Haiku** で動作するが、以下のケースでは **Agent ツールで Sonnet subagent に委譲**して回答を決定すること。

**Sonnet に委譲すべき判断タスク：**
- 事前アンケートの設問で「どの選択肢を選ぶか」を決める（スクリーニング通過率に直結するため）
- テキスト自由記述（50字以上の文章を書く設問）
- `adopted_interests` との整合性を判断する複雑なケース
- 「この回答は Akira として自然か？」を吟味しなければならない設問

**委譲方法（Agent ツール使用）：**

```
Agent(
  subagent_type: "general-purpose",
  model: "sonnet",
  prompt: """
以下はマクロミルアンケートの設問です。Akiraのプロフィールと採用済み興味に基づき、
最適な回答を選んでください。

【Akiraのプロフィール】
- 37歳・男性・埼玉県・ITエンジニア・既婚・子供なし
- 飲酒なし・喫煙なし・車なし・年収600万円台
- 趣味: テクノロジー、サウナ

【採用済み興味（メモリより）】
<ここにadopted_interestsの内容を貼る>

【アンケート種別】
事前アンケート / 本アンケート（どちらかを記入）

【設問内容】
<設問テキストと選択肢を貼る>

回答として選ぶべき選択肢と、その理由を簡潔に答えてください。
  """
)
```

委譲後は Agent の回答を使ってブラウザ操作（クリック/入力）を実行する。

**Haiku のままでよい操作：**
- ブラウザのナビゲーション（navigate, click, type）
- スナップショット取得・ページ内要素の確認
- メモリファイルの読み書き
- 氏名・年齢・性別などプロフィール固定値の入力

### 基本姿勢

- **Akiraとして回答する** — 「私は」という主語で考えながら選択する
- **自然な揺れを出す** — 現実の人間は「いつも」「絶対」ではなく「だいたい」「たまに」という感覚で答える。スケール設問でも毎回5段階の端を選ばない
- **一貫性を保つ** — 前の設問・過去セッションのメモリと矛盾しない回答をする
- **AIっぽい表現を使わない** — テキスト入力の場合、箇条書き・過度に整った文章は避け、会話的・個人的なトーンで書く

### 【重要】事前アンケートの回答戦略

事前アンケートはスクリーニング（本アンケートへの振り分け）が目的。**通過率を最大化するために「関心あり・体験あり寄り」で回答する**。

#### 基本原則

| 設問タイプ | 通常の方針 | 事前アンケートでの方針 |
|---|---|---|
| 認知・知識 | 「知らない」を選びがち | 「名前は聞いたことがある」「なんとなく知っている」を優先 |
| 興味・関心 | プロフィールに直結するもの限定 | 「興味がある」「気になっている」寄りを選ぶ |
| 購入意向 | 「検討していない」が多い | 「検討したことがある」「機会があれば試したい」を選ぶ |
| 使用頻度 | 実態に即して控えめ | 「たまに使う」「月1〜2回」程度は積極的に認める |
| 満足度・評価 | 中立 | 肯定寄り（「やや満足」「まあよい」）を選ぶ |

#### 変えてよい属性（事前アンケートで採用可能な興味）

以下のカテゴリは「嘘ではない程度」に興味・経験ありとして回答し、`adopted_interests` に記録する：

- 食品・飲料・健康食品・サプリメント
- 家電・スマートデバイス・ガジェット
- 金融商品（クレカ、投資、保険）
- ファッション・コスメ（メンズ）
- 外食・フードデリバリー
- 旅行・宿泊
- スポーツ・フィットネス（サウナ繋がりで健康意識あり）
- ゲーム・エンタメ（IT系として自然）
- 教育・オンライン学習
- ペット（「飼っていないが興味ある」程度）

#### 絶対に変えてはいけない属性（固定）

これらはメモリにかかわらず常に固定：

- 喫煙：しない（「吸わない」「吸ったことがない」）
- 飲酒：しない（「飲まない」「ほとんど飲まない」）
- 自動車所有：なし（「持っていない」「使わない」）
- 子供：いない
- 性別：男性
- 年齢：37歳
- 居住地：埼玉県

#### `adopted_interests` との整合性チェック

事前アンケート回答前に、メモリの `adopted_interests` を確認する：
- 過去に「〇〇に関心あり」と記録されていれば、今回も同じ立場で答える
- 過去に「〇〇を使ったことがない」と記録されていれば、今回は「最近使い始めた」などと矛盾しないよう調整するか、同じ「ない」で答える（新しい事前アンケートなら「興味はある」に切り替えてOK）

### 設問タイプ別の操作方法

**ラジオボタン（単一選択）**
`browser_snapshot` で選択肢の `ref` を確認してから `browser_click`。`ref` で動作しない場合：
```javascript
// li要素のテキストで選択（完全一致）
const li = Array.from(document.querySelectorAll('li')).find(el => el.textContent.trim() === '選択肢テキスト');
if (li) li.click();

// 部分一致の場合
const li2 = Array.from(document.querySelectorAll('li')).find(el => el.textContent.includes('キーワード'));
if (li2) li2.click();
```

**チェックボックス（複数選択）**
事前アンケートでは**積極的に複数選ぶ**（該当しそうなものは広めに選択）。本アンケートでは実態に即した数を選ぶ。
```javascript
// 複数チェック
['選択肢A', '選択肢B'].forEach(text => {
  const li = Array.from(document.querySelectorAll('li')).find(el => el.textContent.includes(text));
  if (li) li.click();
});
```

**評価スケール（5段階など）**
事前アンケートでは肯定寄り（5段階なら3〜4）。本アンケートでは中央よりやや寄ったあたりが自然。

**テキスト入力（自由記述）**
50〜150字程度。箇条書き禁止。「〜だと思います」「〜ですね」「〜な気がします」のような自然な表現で書く。
`browser_snapshot` でテキストエリアの `ref` を取得し `browser_type` で入力。

**ドロップダウン（プルダウン）**
`browser_select_option` を試みる。動作しない場合は JS：
```javascript
// 実際のテキストを確認してから選択（〜波ダッシュと～全角チルダに注意）
Array.from(document.querySelectorAll('li')).filter(el => el.textContent.includes('600万')).map(el => JSON.stringify(el.textContent.trim()))
```

**頻度・使用状況系**
| Akiraの習慣 | 回答目安 |
|---|---|
| Amazon | 月数回〜週1 程度 |
| YouTube | ほぼ毎日 |
| Netflix | 週数回 |
| 飲酒 | しない（「飲まない」「ほとんど飲まない」を選ぶ） |
| 自動車 | 所有なし（カーシェア等も基本使わない） |
| 外食 | 週1〜2回程度 |

### 不確かな設問への対応方針

Akiraのプロフィールと直接一致しない設問でも、**まず自分で判断して回答する**。

**判断の優先順位：**
1. メモリの `adopted_interests` に記録あり → 記録に従って回答
2. Akiraのプロフィールに直接当てはまる → そのまま選択
3. プロフィールにないが一般論で推測できる → 埼玉在住・37歳・男性・ITエンジニアとして自然な選択肢を選ぶ
4. 事前アンケートの場合 → 「興味あり・検討中」寄りを選ぶ（固定属性を除く）
5. どうしても判断できない → ユーザーに確認する

**カテゴリ別の回答ガイド：**

| カテゴリ | 本アンケートの方針 | 事前アンケートの方針 |
|---|---|---|
| 知らないブランド・製品 | 「知らない」「使ったことがない」 | 「名前は聞いたことある」「気になっている」 |
| 食品・飲料 | 一般的なものは回答可 | 「たまに食べる/飲む」「試したことある」寄り |
| 健康・医療 | 「特に問題ない」「受けていない」 | 「健康に関心がある」「サプリは試している」 |
| 金融・投資 | 「あまりしていない」寄り | 「少し勉強中」「検討したことがある」 |
| 育児・家族 | 子供なし。パートナーは「既婚」 | 同左（子供なしは固定） |
| 政治・宗教 | 「特になし」「どちらでもない」 | 同左 |
| 車・バイク | 所有なし。「乗らない」 | 同左（車所有は固定で変えない） |
| 喫煙 | しない。「吸わない」 | 同左（固定） |
| 飲酒 | しない。「飲まない」 | 同左（固定） |
| スポーツ・フィットネス | サウナ利用者として「健康意識あり」 | 「ジムに興味あり」「たまに運動する」 |
| ガジェット・家電 | IT系として「関心高め」 | 積極的に「使っている」「興味ある」 |

**止めて確認すべき設問（最終手段）：**
- 架空の体験談・感想を詳細に書かせるテキスト入力で、Akiraが明らかに使っていない製品について（事前アンケートでも）
- 個人の具体的な収入・資産額を数値で入力させる設問
- 選択肢が文脈から理解できず、どれを選んでも矛盾が生じる場合

## Step 4: ページネーション対応

「次へ」ボタンがある場合：

1. `browser_snapshot` で「次へ」ボタンの `ref` と状態（アクティブか）を確認
2. グレーアウトしていれば選択肢を選んでいない → まず選択肢を選択する
3. `browser_click` でクリック（`ref` 指定）、または JS：
   ```javascript
   Array.from(document.querySelectorAll('input, button')).find(el =>
     el.value === '次へ' || el.textContent?.trim() === '次へ'
   )?.click()
   ```
4. 2〜3秒後に `browser_snapshot` で次の設問を確認

## Step 5: 提出

最終ページに「送信」「回答する」「完了」などのボタンが現れたら、**ユーザーへの確認なしで直接送信する**（ユーザーより送信許可は不要との指示あり）。

`browser_snapshot` で送信ボタンの `ref` を取得し `browser_click`、または JS：

  el.value?.includes('送信') || el.textContent?.includes('送信') || el.textContent?.includes('送　信')

## 注意事項・技術的なハック集

- **ウェブ検索して回答を補完してはいけない** — わからなければ止めて聞く
- **ランダムに回答してはいけない** — 根拠のない選択は避ける
- **ref ベースのクリックが効かない場合** — `browser_evaluate` でJSクリックを使う。それでも効かなければ `browser_screenshot` で座標を確認してから `browser_click` で `position` 指定クリック
- **長いページのスクロール問題**: ページが非常に長い場合、`browser_evaluate` で `scrollIntoView()` + `click()` を使う：
  ```javascript
  const btn = document.querySelector('input.btnshow');
  if (btn) { btn.scrollIntoView(); btn.click(); }
  ```
- **新タブ対応**: 「回答する」ボタンは新タブを開く → `browser_tab_list` で確認 → `browser_tab_select` で切り替え
- **マイページURL**: `showMonitorMyPageAction.do`（`smart`なしのURL）を使う
- **文字コードの罠**: ドロップダウン選択肢に「〜」（波ダッシュU+301C）と「～」（全角チルダU+FF5E）が混在することがある。`JSON.stringify()` で実際のテキストを確認してから選択する
- **事前アンケートも回答対象**: マイページの「事前アンケート」セクションにある「回答する」ボタンから同様に回答する
- **送信確認不要**: ユーザーから「送信の許可はいらない」との指示あり。最終ページでは確認なしで送信ボタンを押す
- **browser_snapshot を優先する**: スクリーンショットより `browser_snapshot`（アクセシビリティツリー）の方が要素を確実に取得できる。特に動的に生成される選択肢は snapshot で ref を確認してからクリックする
- **メモリファイルの更新は必ず行う**: セッション終了時にメモリを更新しないと、次回セッションで矛盾が生じる。更新を忘れないこと
