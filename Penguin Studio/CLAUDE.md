# CLAUDE.md — Penguin Studio

## プロジェクト概要

Penguin Studio の案件作業ワークスペース。クライアント・社内案件ごとにディレクトリを分けて管理し、機密情報を含む各種ドキュメント処理・作業の自動化・スキル化を行う。

## Claudeの役割と行動指針

**主な役割:**
- 各案件の作業を自動化・スキル化する
- ドキュメント処理（変換・整理・作成）
- 案件ごとの成果物をそのディレクトリ内に配置する

**自律度:**
- 基本は自分で判断して実行する
- ファイル作成・編集・スクリプト実行は確認不要
- 不可逆な操作（削除・外部送信等）は必ず確認する

**やらないこと:**
- 機密情報を外部ツール（WebSearch・外部API等）に送信しない
- git 管理対象の案件ディレクトリへのコミット（.gitignore で除外済み）

## ディレクトリ構成

```
Penguin Studio/
├── CLAUDE.md              ← このファイル
├── .gitignore             ← 案件ディレクトリを git 管理外に
├── pptx-converter/        ← ツール（git 管理対象）
├── [案件名]/              ← 案件ごとのディレクトリ（git 管理外）
│   └── [成果物・入力ファイル]
└── *.skill                ← スキルファイル（git 管理対象）
```

**命名規則:** 案件ディレクトリは取引先名または案件名（例: `TOPPAN/`, `○○株式会社/`）

## 機密情報の取扱い

- 案件ディレクトリは `.gitignore` で git 管理外
- ワークスペース内での参照・処理のみ行う
- 成果物は必ず該当案件ディレクトリ内に配置する
